Contributing

We encourage everyone from Home Office Digital to join the discussion on the design patterns.

1. Talk about it (crit)

Crits are a good place to discuss work with other designers to help solve design issues. A larger crit session is organised once every two weeks but informal crits are recommended whenever the need arises, they should include at least 3 designers.

  • If possible, share the designs before a crit
  • Clarify the user need
  • Don't view the element in isolation, understand the journey
  • Explain the design issue

2. Document it (on the patterns trello board)

If the user need or design solution is new and there's no pattern that resolves the issue, add it to the patterns board and we will review with you so it can be developed to help others.

  • Include what the problem the pattern is trying to solve
  • Add any comments or feedback from the crit
  • Use screenshots, with any personal data redacted

How designs will be reviewed and fed back on.

3. Develop it (and include in the pattern library)

  • Add to the sketch template file, if it's relevant
  • Add to the code repository

4. Share it, let people know the changes

Add some detail around what the pattern does, and let others know (preferrably through slack) that a new pattern has been added.
